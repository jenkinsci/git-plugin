Defines a Git repository reference ("ref") build parameter, enabling selection from a dynamic list of a remote Git repository's head and tag refs. If you wish to see what a ref looks like, or see the list of refs for a given Git repository, you can use the following Git command: 

    git ls-remote http://www.kernel.org/pub/scm/git/git.git

The resulting list of SHA-1 & ref tuples indicates how ref values appear, i.e.: 

    fc98289584956de9ab1402a4cd607415926f5e68        refs/tags/v1.7.11.4

This parameter is particularly useful in conjunction with Git Source Code Management by way of parameterizing the repository refspec, thus controlling which head or tag is built. For example, if Name is set to GIT_REF, its value can be utilized in each build by setting the Git repository Refspec to +${GIT_REF}:refs/remotes/origin/master.
